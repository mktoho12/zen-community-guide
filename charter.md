# ZEN 大学 コミュニティ・ガイドブック プロジェクト憲章

_（本プロジェクトの目的と統治構造を定義する最上位文書であり、以下「本憲章」と呼ぶ。具体的な運用細則は「運用規約」に別途定める。）_

---

### 第 1 章：定義と構成員

**本プロジェクト**
本プロジェクトは、ZEN 大学の Slack コミュニティ内に存在する多様な活動を「発見」するための「文脈付きガイドブック」を構築・運営する、学生による独立した活動である。

**構成員**
本憲章における構成員は、以下の役割（ロール）で定義される。

- **創設者 (Founder):** 本プロジェクトの立案者（えむけー）。
- **レビュアー (Reviewer):** 掲載内容の承認、非掲載、削除の判断を行う合議体。
- **編集者 (Editor):** 掲載内容（紹介文）の作成・修正を申請する協力者。
- **利用者 (User):** 本ガイドブックを閲覧する全ての学生・関係者。

### 第 2 章：目的と基本原則

**目的**
本プロジェクトの唯一の目的は、ZEN 大学の学生が、自身に最適な学習・交流・制作活動への **「発見アクセスコスト」** を引き下げることにある。

**基本原則**
本プロジェクトは、目的達成のため以下の原則を遵守する。

1. **序列化の否定:** 本ガイドブックは、チャンネルの「人気」や「活発さ」に基づく **ランキングやスコアリングを一切行わない** 。
2. **非序列メタ情報の許容:** 上記 1 項に関わらず、利用者の「探索効率」を高めるための **非序列的なメタ情報（タグ、活動頻度、雰囲気、参加コスト等）** の付与は、これを積極的に行う。
3. **統治の独立:** 本プロジェクトは、大学当局、特定サークル、その他外部団体の管理下には入らず、本憲章に基づき学生が自律的に運営する。

### 第 3 章：統治（ガバナンス）設計

**統治体制**
本プロジェクトの意思決定は、本憲章および「運用規約」に基づき、創設者、レビュアー、編集者によって実行される。

**創設者の暫定権限 (Temporary Founder Authority)**

1. **権限内容:** 最初のレビュアー体制が確立するまでの間、掲載・非掲載・修正の最終決定権、および本憲章の「軽微な修正」権を持つ。「軽微な修正」とは、表記ゆれなど、本憲章の統治構造に影響を与えない変更を指す。

2. **義務:** 権限行使の際は、そのすべての決定と理由を「運用ログ」に記録し、透明性を担保する。

3. **終了条件:** 本権限は、あらかじめ定められた「終了条件（プロジェクト開始後 3 ヶ月、またはレビュアーが 3 名任命された時点のいずれか早い方）」をもって自動的に失効し、全権限をレビュアー合議体に譲渡する。

**権力移譲計画 (Power Transition Plan)**
本プロジェクトは創設者への「個人依存」を脱却するため、以下の計画に沿って統治を非個人化する。

1. **Phase A (0-3 ヶ月):** 創設者による暫定統治。
2. **Phase B (3-6 ヶ月):** 初期レビュアー（3 名）を選定し、合議制に移行する。
3. **Phase C (6 ヶ月以降):** 第３章「学生自治フェーズへの移行」に基づき、完全な「学生自治」フェーズに移行する。

**レビュアーの任命手続**
レビュアーは、コミュニティからの信頼に基づき、「運用規約」に定める手続き（募集期間、応募資格、選定基準、任期）に従って任命される。

**学生自治フェーズへの移行**
Phase C 移行後、本プロジェクトは以下の 3 つの責任ロールを選任し、創設者が不在でも運営可能な体制を構築する。

1. **技術管理:** インフラ（GitHub, ドメイン等）を管理する。資産は創設者個人名義からプロジェクト（または技術管理ロール）へ速やかに名義を移管し、属人性を排除することを目標とする。
2. **編集管理:** レビュアーの任命、編集フローの全体を監督する。
3. **外部対応:** 大学当局や他サークルとの公式な窓口となる。

### 第 4 章：独立性の担保

**利害の境界**
本プロジェクトの独立性を守るため、外部との境界を以下の通り定義する。

1. **チャンネル運営者:** 紹介文の修正を「提案」できる。最終判断はレビュアーが行う。
2. **大学当局:** 本ガイドブックの「内容（編集権）」に介入することはできない。ただし、「運用規約」に定めるプロセスに基づく「削除・修正要請」は受け付ける。

**非公式の明示 (Disclaimer)**
本ガイドブックは学生有志による非公式の活動であり、ZEN 大学の公式見解ではない。この**免責事項**は、ガイドブック上の**「全てのページ」**に、フッター等を通じて恒久的に明記されるものとする。

### 第 5 章：憲章の改定と優先順位

**憲章の改定**
本憲章の改定は、本プロジェクトの根幹に関わる重大な意思決定である。改定の発議は構成員なら誰でも可能だが、承認には「レビュアーの 3 分の 2 以上」の賛成を必要とする。

**文書の優先順位（正文）**

1. **優先順位:** 本憲章は、すべての「運用規約」および「運用ログ」に優先する最上位の文書である。
2. **正文:** 本憲章は、日本語版の最新版（GitHub リポジトリで管理されるもの）を「正文」とする。他言語版や短縮版（スライド等）との間に矛盾が生じた場合は、「正文」の記述が優先される。
