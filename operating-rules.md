# ZEN 大学 コミュニティ・ガイドブック 運用規約

_（本規約は、「プロジェクト憲章」に基づき、ガイドブックの具体的な運用・編集・対応プロセスを定める実施細則である。本規約は、実務上の必要性に応じ、「レビュアーの過半数」の賛成により改定できる。）_

---

### 第 1 章：窓口と申請プロセス

**1.1 公式連絡窓口**
本プロジェクトへの一切の公式な連絡（掲載申請、削除要請等）は、 **「GitHub リポジトリの Issue Tracker」** を公式窓口とする。

1.  **例外対応:** GitHub アカウントを持たない等の理由で公式窓口が利用できない場合、他の手段（メール等）による連絡を仮受付するが、その内容は運営側で公式窓口（Issue）に転記・公開し、対応を一元管理する。
2.  **DM 等の非公式対応:** Slack DM 等、公式窓口以外での依頼は、正式な申請とはみなされず、公式窓口への再送を要求する。

**1.2 MVP（最小実行可能プロダクト）の定義**
本規約における「MVP」公開とは、以下の 2 条件をすべて満たした状態を指す。

1.  第 2.2 条に基づく「公開ログ」が技術的に作動していること。
2.  第 2.1 条のテンプレートに基づく掲載が **3 件以上** 公開されていること。

**1.3 MVP 公開前の申請**
MVP 公開前に受理した掲載申請は、創設者（またはレビュアー）の裁量に基づき、公開ログに記録の上で「保留」することができる。

### 第 2 章：編集プロセスと透明性

**2.1 編集の型（テンプレート）**
掲載情報の品質と公平性を担保するため、すべての紹介文は以下のテンプレートに沿って作成される。

- 活動の目的（100 字以内）
- 主な参加者像（学年・興味関心）
- やり取りの頻度・トーン（静か／日常雑談／高速など）
- 参加コスト（入るだけ／事前申請／企画参加が必要）
- 初回参加者への具体的導線（最初に読むスレ・自己紹介など）

**2.2 公開ログの場所と管理**
すべての意思決定ログは、 **「GitHub リポジトリ」** を一次記録（Source of Truth）として恒久的に公開する。
ログの登録実行は、Phase A/B においては「創設者」が、Phase C 以降は「編集管理」または「技術管理」ロールが責任を持って行う。

### 第 3 章：権利と責任

**3.1 著作権とライセンス**

1.  **ライセンス許諾:** 編集者は紹介文を投稿した時点で、当該テキストを **「CC BY-SA 4.0」** ライセンスの下で提供し、本プロジェクトがそれを改変・頒布するための **取消不能な権利** を許諾したものとみなす。
2.  **例外的措置:** 原則(a)に関わらず、「レビュアー合議」の決定に基づき、以下の「例外的措置」を講じることができる。
    - **重大な権利侵害（個人情報等）:** 発見次第、直ちに非公開または修正する。
    - **本人からの撤回希望:** 投稿から **30 日以内** の申請に限り、原則として当該記述の非公開または修正を行う。

**3.2 違反時の扱（スリーストライク制）**

1.  **スリーストライク制:** 憲章の原則に反する申請が確認された場合、レビュアーは警告を行う。警告が **3 回** 蓄積した場合、対象者は「ブラックリスト」に追加される。
2.  **ブラックリスト:** ブラックリストに追加された「申請者」および「対象チャンネル」からの以降の申請は、 **原則として拒否する** 。
3.  **再審制度:** ブラックリストの有効期間は **6 ヶ月** とする。対象者は期間中 1 回に限り **「異議申し立て」** を行える。また、(b)項の原則に関わらず、レビュアーの過半数が必要と認めた場合、例外的に申請を受け付けることができる。

**3.3 予算・金銭**
本プロジェクトの活動は無償のボランティアである。資金が確保された場合、その原資と使途は「会計報告」として公開する。

### 第 4 章：インシデント対応

**4.1 インシデント対応の優先順位**
問題発生時、レビュアーは以下の優先順位に基づき判断する。

1.  **最優先：当事者の安全（Safety）**
2.  **次点：法令・大学規程の遵守（Compliance）**
3.  **三番手：透明性の確保（Transparency）**
4.  **四番手：ガイドブックの可用性（Availability）**

**4.2 削除・修正要請プロセス**

1.  **仮受付（非公式要請の扱い）:** 口頭、電話、DM 等、公式窓口以外からの要請は **「仮受付」** とし、 **公式窓口からの記録可能な手段による再送** を要求する。
2.  **段階 1：緊急一時非公開:** 正式な要請を受領した場合、レビュアー（または創設者）は、調査のため直ちに当該掲載を **「一時非公開（凍結）」** にする権限を持つ。
3.  **段階 2：内部レビューと事後ログ:** レビュアーは非公開の理由を審査し、「完全削除」「修正の上再掲載」「要請棄却」を合議で決定する。この決定プロセスと最終結果は、公開ログに記録する（個人情報等に関わる詳細は概要のみ）。
